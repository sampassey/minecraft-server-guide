How to Set Up a Local
Minecraft Server with Docker

A step-by-step guide to hosting your own Minecraft Java server for free

Introduction

When hosting your own Minecraft world, one of the biggest frustrations is that your friends can only play when you, the host, are signed on and in the world. Setting up a dedicated server solves this problem by keeping the world available at all times, no matter who is online. These instructions show how to create your own server, available 24/7 and completely free.

To complete these instructions, experience executing commands from your terminal is helpful, but not required. For beginners interested in understanding each terminal command or the big-picture concepts, optional explanatory detail has been provided. On the other hand, if you already have worked with terminal and are just looking to get a Minecraft server up-and-running as fast as possible, you should be able to scan through and complete these instructions in just a few minutes.

By the end of this tutorial, you will have your own Minecraft server to play on with your friends.

While a Minecraft server can be hosted in a number of different ways, this document explains how to set up and run a server on a physical device running Linux (such as a Raspberry Pi, or spare laptop with Ubuntu) or on a Mac. The server will be available for anybody on your local network to join, provided they have their own copy of Minecraft Java Edition.

Setting up a server you could join from anywhere in the world is beyond the scope of these instructions, as doing so comes with a jump in complexity, security concerns, as well as potential hosting costs. However, learning how to host a local server can provide the familiarity with the foundational tools, concepts, and terminology for hosting a more sophisticated server on the web in the future.

Most of the work for setting up the server will be done on the terminal and the server itself will run on software called Docker. While macOS, Windows, and Linux all are capable of running Docker, these instructions focus on the methods of installing Docker that are specific to Linux and macOS only.

The instructions will illustrate how to install Docker, configure the server, run the server, and shut down the server using the terminal. There is also a list of precautions and warnings provided to help you run the server safely and successfully. You will also find a few troubleshooting tips at the end of the instructions to help if something does not appear to be working right.

Requirements

OS
Linux (Ubuntu Server, Debian, or Raspberry Pi OS 64-bit) or macOS (macOS 12–14 on Apple Silicon).
RAM
4 GB minimum required for smooth multiplayer gameplay on a Java Minecraft server.
Network
Local network connection via Wi-Fi or Ethernet (Ethernet recommended for stability).
Device
A dedicated machine is recommended, since running a server can use significant resources and make other tasks difficult.
Note Windows is not supported in these instructions. See the Minecraft wiki for Windows setup.
What is Docker?
Docker is a lightweight software that packages an app (such as a Minecraft server) and everything the app needs to run into a standardized, isolated unit called a container. A Docker container runs consistently across different computing environments, hardware, and operating systems, from a simple laptop to a powerful server computer.
Safety and Precautions
  • Network exposure. This setup is designed for a local network only. Exposing ports directly to the public internet without proper safeguards can compromise not just your server, but potentially all devices connected to your network.
    • Safe methods to connect the server to the broader internet exist, but fall outside the scope of these instructions.
  • Keep things updated. Keep your device's OS and Docker installation updated to reduce the likelihood of failure and vulnerability.
    • Refer to Docker's online documentation for device and installation-specific ways to update the software.
  • Back up your world. Regularly back up the Minecraft data folder (minecraft-java-data) to another storage location or external drive.
    • That way you can recover your worlds and configuration if the device fails or storage becomes corrupted.
  • Hardware limits. Avoid using a personal device to host a large server.
    • The server described in these instructions is best used in small, local network applications with up to 10–15 players.
    • Limited CPU and RAM may cause lag or crashes with too many players, plugins, or mods.
  • License compliance. Only use the server in ways that comply with the official Minecraft license.
1

Open a Terminal and Log In

Open the terminal on the machine you wish to install the server on.

When terminal is opened, it will look something like this.

Open a terminal directly on your Linux device by launching the terminal application from the system's desktop or console interface. For most devices, this can be done with Ctrl+Alt+T.

No keyboard or screen?

If your Linux device does not have a keyboard and screen, you can connect to it remotely using SSH (Secure Shell). SSH is a way to connect to a computer and execute commands on it by typing them into your own computer.

From another machine on the same network, use a device with SSH installed to connect to the Linux device; macOS, most Linux systems, and Windows should have this command line tool installed by default.

Open a terminal on another machine (for Linux use Ctrl+Alt+T, for Mac see below, and for Windows press Windows Key+R, type cmd, then press Enter) and run:

Terminal 
ssh <username>@<device-ip>

Replace <username> with your Linux account name, and <device-ip> with the server device's local IP address.

Note If you haven't identified the server device's IP address yet, skip ahead to Step 7 to learn how to find it directly on the device itself.

Once connected, run all commands in the remote SSH terminal session, as the rest of the process is the same as working directly on the device.

  • Open Finder
  • Scroll down until you find Utilities, and open the folder
  • Launch Terminal

OR

  • Open Spotlight Search (Cmd + Space)
  • Type Terminal
  • It will be the top result (Terminal.app)
2

Install Docker

Once you have terminal opened, enter the following into the terminal to download Docker's install script:

Terminal 
curl -L https://get.docker.com -o get-docker.sh
Breakdown
  • curl is a tool that pulls content from a web address. The -L option says, "If you get redirected when you go to this website, go where you're sent."
  • https://get.docker.com is the URL that curl goes to.
  • The -o option tells curl to save the script to a file named get-docker.sh in the folder you run this command in.

After the script finishes downloading, enter this line in terminal to run the script that installs Docker:

Terminal 
sudo sh get-docker.sh
Breakdown
  • sudo tells the terminal to run whatever comes next with administrator-level permissions; you need this to install software.
  • sh tells the terminal to execute a script.
  • get-docker.sh is the script you just downloaded to install Docker to the device.

Once the script finishes executing, Docker will be installed and ready to configure.

  1. Head over to docs.docker.com/desktop/setup/install/mac-install/ and download the package.
  2. Once it's downloaded, double click the disk image, then drag and drop it into the Applications folder.
  3. Open the application through Finder's Application folder OR through Spotlight Search (Cmd + Space) by typing Docker.
  4. The Docker engine will start up and ask for permissions. You'll need to give the app the permissions it requests.
  5. Once the app has been opened and permissions granted, you can do the rest in the terminal. Close the app.
Note on Docker
Docker runs in the background on your computer. Background applications may not have any indication they are running, but they are still consuming your system's resources. This behavior is useful to keep your server running while doing other things, but it is important to be aware of what is running and when. After you start Docker, if you do not want it continuously taking up resources on your device, make sure you turn it off by following the instructions in Step 9 when you are done using it.
3

Create a Docker Project Folder for the Minecraft Server

Now enter the following in the terminal:

Terminal 
mkdir ~/minecraft-docker; cd ~/minecraft-docker

This line creates a folder to store Docker information for the Minecraft server. It keeps the server's files organized, and makes it easy to back up or move the server to a different device later.

Breakdown
  • mkdir is the command for "make directory." A directory is the same thing as a folder. This command creates a directory named minecraft-docker in your device's home folder, denoted by the tilde (~).
  • You can specify any location for this folder that you would like after the mkdir command, but you will use the home folder here for simplicity's sake.
  • If you name it something else, use that directory's path, which you can find using pwd (print working directory) every time you see ~/minecraft-docker.
  • cd is the command for "change directory." This changes your current working directory in the terminal to that folder, so all the following changes will be made there. You can tell which directory you are currently in by looking at the end of the terminal prompt, where you type things in. At this point it should end with ~/minecraft-docker$.
4

Create the docker-compose.yml File

After you have created the minecraft-docker project folder, enter the following into the terminal:

Terminal 
nano docker-compose.yml

An empty nano window will appear:

Breakdown
  • nano is the command to start nano: a terminal-based text-editing software that comes standard with most Linux distributions and macOS.
  • docker-compose.yml is the name of the file you'd like to create. nano will make the file in the terminal's current directory.
  • In Step 3, you changed the current directory to ~/minecraft-docker, so this is the location of your docker-compose.yml file.
What's YAML?

YAML files are just specially formatted plain text files. Programs will often use these to store settings. They're favored because they're easy to read and write.

Docker requires instructions that are provided by specialized text files known as YAML (.yml) files. This code creates one of those text files, which, in the next step, will be loaded with configurations for Docker on how to run the server.

These configurations can be adjusted to change the settings of your server, but these instructions will provide the standard boilerplate code to get a simple server started.

5

Define the Minecraft Server

Copy and paste the following code into the empty nano window to define a standard Java Minecraft server:

docker-compose.yml 
services:
  minecraft-java:
    image: itzg/minecraft-server
    ports:
      - "25565:25565"
    environment:
      EULA: "TRUE"
    volumes:
      - ./minecraft-java-data:/data
    restart: unless-stopped
Indentation matters! YAML files are whitespace-sensitive. Match the spacing exactly as shown above.
What does each line do?
  • services: organizes a list of the containers to run.
  • minecraft-java: is the name of the service being defined. You could name this differently to differentiate between other server images if you make more than one.
  • image: itzg/minecraft-server tells Docker to pull the Minecraft Java server image from Docker Hub. This image file has everything Docker needs to run the server: Java, startup code, and other server instructions. It has been made publicly accessible to Docker users.
  • ports: 25565:25565 publishes the server's Java port (25565) on the device so clients on the local network can connect. In other words, this defines a location that can be found by other players when connecting to the server.
  • environment: is the section for environment variables that the container will use.
  • EULA: "TRUE" sets Minecraft's EULA (End-User License Agreement) to be accepted. Without this, the server will refuse to start.
  • volumes: is the section where Docker is told where to look for relevant data.
  • - ./minecraft-java-data:/data maps the minecraft-java-data folder on your device to the /data directory inside the container. This is where the server stores world data, server properties, and other files.
  • restart: unless-stopped tells Docker to automatically restart the container if it crashes or the device reboots. You can change this to no if you'd like it to only start up when you initiate it manually.

After pasting the content, save the file:

  1. Press Ctrl+X to exit the file
  2. Press Y to accept the changes, when prompted
  3. Press Enter to finalize all changes and exit

Successfully exiting will put you back into a normal terminal interface.

6

Start the Minecraft Server

To run the Minecraft Java server container in the background on your device, type the following command in the terminal:

Terminal 
sudo docker compose up -d
Terminal 
docker compose up -d

On the first run, Docker will automatically download the itzg/minecraft-server image from Docker Hub, if it is not already on the device. Once the image is pulled, Docker will create a container, download the Minecraft server files, and start the server process.

This may take a moment. This command can take anywhere from a few seconds to a few minutes to run. Wait for the process to complete and do not interfere with the terminal during this time.
Breakdown
  • sudo runs the command with administrator privileges (Linux only).
  • docker compose tells Docker to look for the compose file created in Steps 4–5.
  • up starts the service defined in docker-compose.yml.
  • -d runs the container in detached mode (in the background), so you can keep using the terminal.

Verify the Server is Running

Once the previous command finishes, enter the following into terminal to make sure that the Minecraft server is running:

Terminal 
sudo docker ps
Terminal 
docker ps

If you've followed the directions correctly, you should see a row showing the container itzg/minecraft-server, along with other information, like ports, the container name, etc.

Breakdown
  • ps is "process status," and it prints off all running Docker processes.
7

Find Your Server's IP Address

Type the following into the terminal:

Terminal 
hostname -I
  • hostname -I prints the network IP addresses of the device.
Terminal 
ipconfig getifaddr en0
  • ipconfig getifaddr en0 prints the local network IP address of your Mac's primary network interface.

Write down the IP address or copy/paste it somewhere for future reference.

What's an IP address?
You'll need to find your server's address for your friends to join before play. The address of your server consists of two elements: the IP address of the device your server is running on, and the port number 25565 (as defined in Step 5). The device IP address is your computer's address on the network, usually represented by a string of numbers, like 192.168.1.100.
What's a port?
A port is like a virtual door on your computer that directs network traffic to a specific app. Port 25565 is the default port for Minecraft servers.
8

Join the Server From Your Local Network

On a separate device running Minecraft Java Edition on the same network:

  1. Open Minecraft Java.
  2. Click Multiplayer.
  3. Click Add Server.
  4. In the Server Address field, type the IP address of your device followed by :25565
    (e.g., 192.168.1.100:25565).
  5. Click Done, then Join Server.

If everything was done correctly, the account should connect to your locally hosted Minecraft server over the same network. Your Minecraft character is now alone in the wilderness, ready to find your friends and have an adventure!

9

Shutting Down the Server

To stop the server while keeping its configuration and data, enter the following into the server device's terminal to shut the server off without risking any loss of data in your Minecraft world:

Terminal 
cd ~/minecraft-docker; sudo docker compose stop
Terminal 
cd ~/minecraft-docker; docker compose stop
Note If you are hosting the server on your personal computer, it is good practice to shut it down when you are done playing. Allowing a busy server to continue running in the background can consume a significant amount of your machine's computational resources.
Breakdown
  • cd ~/minecraft-docker changes the current directory to the location of the docker-compose.yml file. This should be the same directory you set the server up in, so if you changed it there, change it here too.
  • (sudo) docker compose stop tells the docker-compose.yml file in the current directory to stop all Docker container operations, including the active Minecraft server.

Starting it back up

To turn the server back on, enter the following into the terminal:

Terminal 
cd ~/minecraft-docker; sudo docker compose start
Terminal 
cd ~/minecraft-docker; docker compose start

Additionally, you can turn the server back on by restarting your device. (Recall that in Step 5, the restart variable was set to restart: unless-stopped.)

Breakdown
  • (sudo) docker compose start tells the docker-compose.yml file in the current directory to activate all Docker container operations, including the Minecraft server.

Conclusion

Congratulations! You've successfully created and tested your own Minecraft server! Celebrate by inviting friends to join and enjoy your world together. If you'd like to expand your server's capabilities, try exploring options like:

  • Setting up a Tailscale VPN to give your server a secure, shareable IP address for remote access.
  • Downloading a pre-made world, like the Uncensored Library.
  • Adding mods to your server to customize gameplay and introduce new features.
  • Automating server backups to protect your world and keep progress safe.

Or feel free to keep the server as it is. Regardless, you have now set up your own Minecraft server and learned some technical skills along the way!

Troubleshooting

General Advice The most important piece of troubleshooting advice is to be exact. The commands, the configuration .yaml file, and instructions must be followed exactly. There is no "close enough" in this process!

Nothing appears when you check if the server is running (Step 6)

Check that you are in the ~/minecraft-docker directory and rerun:

Terminal 
sudo docker compose up -d
Terminal 
docker compose up -d

If this does not work, check to see if the compose file has an error. Open the docker-compose.yml file using this command and check that your indendation matches:

Terminal 
nano docker-compose.yml

Docker itself is not working ("command not found")

Verify Docker is installed and running by re-running the Docker install script (Step 2) and then trying the docker ps command again (sudo docker ps on Linux). Return to Step 6 and repeat the startup and verification steps until docker ps shows a running minecraft-java container.
